# Security Policy

SIGIL is primarily a research and tooling repository, not a hosted service, but
security issues still matter.

## Reporting a vulnerability

Please do not open public issues for security-sensitive problems.

Instead:

- use GitHub's private vulnerability reporting flow if it is enabled for the repository
- or contact the maintainer privately through the repository owner's public contact channel

When reporting, include:

- affected file or component
- impact
- proof of concept or reproduction steps
- suggested mitigation if you have one

I will acknowledge valid reports as quickly as possible and coordinate a fix
before public disclosure when appropriate.
